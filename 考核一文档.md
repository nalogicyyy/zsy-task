### 二、GitHub 是干啥的？Pull Request、Fork 是啥意思？  

- **GitHub**：大家把自己写的代码存在上面，既能随时备份，还能邀请别人一起改代码、一起做项目，全世界的开发者都能在上面互相学习、贡献代码。  
- **Fork**：可以将别人写的代码复制下来，之后自己这份上随便改，都不会影响原来的代码。  
- **Pull Request（PR）**：自己发现别人的项目还有可优化的地方后，就可以给原作者发个请求，询问他是否愿意采纳。  
- 我是从哪学的：看 GitHub 官方的新手教程、在B站上跟着博主学习。

### 三、打开 GitHub 很慢/打不开的经历
 
我也遇到过打开 GitHub 特别慢，甚至半天刷不出来的情况。  
**原因主要是**：GitHub 的服务器在国外，国内网络访问时比较困难，有时网比较差。    
**解决方法**：下载合适的加速器。

### 四、这份文档为啥算整洁规范？  

我是照着这些 Markdown 规则写的，所以看起来干净清楚：  

* 使用标题标题：一眼就能找到想看的部分。  
* 列清单：用  *  把要点一条条列出来，不用读大段文字，信息更直观。  
* 画分割线：用  ***  把不同主题隔开，避免内容堆在一起，看着更舒服。  
* 标重点：像  Pull Request  这种专业词，标出重点。
